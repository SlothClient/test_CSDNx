## 引言：当代码开始摸鱼时
各位在座的代码劳工朋友们，今天我们要探讨一个令人振奋的哲学命题——如何让我们的函数像聪明的打工人一样学会"战略性躺平"。这不是偷懒！这是人类智慧与机器逻辑的完美融合（一本正经地胡说八道）。

想象一下：你的函数第一次执行时突然原地变身，对着后续的调用者邪魅一笑："后面的兄弟不用再查啦，答案我都记在小本本上了"。这种风骚的操作，就是我们今天要解剖的"惰性函数"。

---
## 第1章 函数界的变形记
### 1.1 什么是惰性函数？
简单来说，这类函数会在第一次执行时：
1. 完成既定任务
2. 悄悄修改自己的DNA（函数体）
3. 下次执行时直接使用优化后的形态

就像程序界的曼德拉草，被调用时会发出尖叫，然后永久改变自己的属性（哈利波特迷请在此处会心一笑）。
### 1.2 经典场景重现

```javascript// 浏览器特性检测的经典姿势function getScrollTop() {    if ('pageYOffset' in window) {        // 检测完毕立即改写自己！        getScrollTop = () => window.pageYOffset    } else {        getScrollTop = () => document.documentElement.scrollTop    }    return getScrollTop()}```
（此时函数内心OS：检测一次就够了，我可不想当复读机！）

## 第2章 那些年我们写过的"真香"代码
### 2.1 事件绑定的自我救赎

```javascriptfunction addEvent(element, type, handler) {    if (window.addEventListener) {        // 检测到标准方法后立即黑化        addEvent = (el, t, h) => el.addEventListener(t, h)    } else if (window.attachEvent) {        // IE遗老专用通道        addEvent = (el, t, h) => el.attachEvent('on' + t, h)    } else {        // 原始人操作方式        addEvent = (el, t, h) => el['on' + t] = h    }    // 记得执行首次调用    addEvent(element, type, handler)}```
（函数内心：第一次总要麻烦点，以后就可以做快枪手啦！）
### 2.2 防抖函数的量子态

```javascriptfunction debounce(fn, delay) {    let timer = null    // 执行后立即升级为完全体    const upgraded = (...args) => {        clearTimeout(timer)        timer = setTimeout(() => fn(...args), delay)    }    // 偷梁换柱大法    debounce = () => upgraded    return upgraded}```
（此时debounce函数露出了蒙娜丽莎般的微笑）

## 第4章 新时代的瑞士军刀
### 4.1 Proxy实现优雅躺平
```javascriptfunction createLazy(fn) {    let upgraded    return new Proxy(fn, {        apply(target, thisArg, args) {            if (!upgraded) {                const result = target.apply(thisArg, args)                upgraded = (...args) => result                return result            }            return upgraded.apply(thisArg, args)        }    })}```
（Proxy：没想到吧，我才是幕后黑手！）
