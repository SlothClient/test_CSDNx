## 1. 代码功能解析（人类语言翻译版）
```javascript
// 灵魂画手注释版
function getScrollTop() {
    // 阶段一：侦察兵模式
    if ('pageYOffset' in window) {
        // 发现现代浏览器特征！
        // 立即把自己变成快捷通道
        getScrollTop = () => window.pageYOffset
    } else {
        // 检测到IE等老古董
        // 切换为兼容模式
        getScrollTop = () => document.documentElement.scrollTop
    }
    // 立即执行新版本函数（重要！）
    return getScrollTop()
}
```

**核心功能**：获取当前页面垂直滚动距离，根据浏览器环境自动选择最优解方案，**且只检测一次浏览器特性**

function getPi() {

## 3. 实现步骤拆解（手把手教学）

### 步骤一：创建侦察兵
```javascript
function scout() {
    // 这里放置需要初始化的检测逻辑
    if (/* 某种条件 */) {
        // 条件成立时的形态
        scout = () => /* 简化操作A */
    } else {
        // 条件不成立的形态
        scout = () => /* 简化操作B */
    }
    // 立即执行新函数（关键步骤！）
    return scout()
}
```

### 步骤二：函数自残（划掉）自我改造
| 操作          | 代码示例                     | 作用                      |
|-------------|--------------------------|-------------------------|
| 检测环境        | `if('xx' in window)`     | 判断浏览器特性                |
| 函数重新赋值      | `scout = () => ...`      | 修改函数本体                 |
| 立即执行新版本     | `return scout()`         | 保证首次调用也能返回正确结果          |

### 步骤三：享受成果
```javascript
// 首次调用：耗时10ms（检测+计算）
scout() 

// 第10086次调用：耗时0.1ms（直接返回结果）
scout() 
```

## 4. 常见翻车现场（避坑指南）

### 4.1 闭包陷阱（经典错误）
```javascript
function badExample() {
    const data = new Date() // 这个值会被锁定！
    badExample = () => console.log(data)
}

badExample() // 输出：2023-07-20
badExample() // 仍然输出：2023-07-20（不是当前时间！）
```

**原因**：原始函数的闭包作用域被锁定，重写函数无法更新闭包变量

### 4.2 动态环境灾难
```javascript
// 在浏览器窗口大小变化后...
getScrollTop() // 仍然使用首次检测的方案，可能导致错误！
```

**黄金定律**：只有在**检测条件永不改变**的场景才能使用！
