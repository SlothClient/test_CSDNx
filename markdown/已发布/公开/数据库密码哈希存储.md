### 关于密码哈希存储

在构建登录注册系统时，安全性是最重要的考量之一。直接将用户的明文密码存储在数据库中是不安全的，因为一旦数据库泄露，用户的密码就会暴露。为了解决这个问题，我们可以使用密码哈希（加密）存储方式。

#### 为什么要使用哈希存储密码？
哈希是一种不可逆的加密方式，即通过算法加密后的结果无法直接还原成原来的数据。在本案例中，我们使用 PHP 的 `password_hash` 函数来加密密码，并通过 `password_verify` 函数来验证用户的输入密码是否匹配数据库中的哈希密码。

#### 使用 `password_hash` 和 `password_verify` 函数
`password_hash` 是 PHP 提供的一个内置函数，专门用于安全地存储密码，它默认使用 **bcrypt** 算法来生成强度较高的哈希。这样就算数据库泄露，攻击者也无法轻易破解密码。

```php
// 注册时使用 password_hash 来生成哈希密码
$hashedPassword = password_hash($password, PASSWORD_DEFAULT);
```

在注册时，我们用 `password_hash` 将用户输入的明文密码加密后存储。然后在用户登录时，使用 `password_verify` 对比输入的密码和数据库中的哈希密码是否匹配：

```php
if ($user && password_verify($password, $user['password'])) {
    // 登录成功
}
```

#### 哈希存储的优势
1. **防止数据库泄露风险**：即使数据库被泄露，攻击者也无法直接拿到明文密码。
2. **内置盐值**：`password_hash` 会自动生成一个随机的盐值（salt），进一步提升安全性，防止彩虹表攻击。
3. **可移植性**：`password_hash` 提供跨平台的安全性，使代码能够在不同环境中运行。

#### 小结
通过 `password_hash` 和 `password_verify` 的方式存储密码，我们不仅保护了用户的敏感信息，还大大提高了系统的安全性。密码哈希存储是所有现代 Web 应用必备的安全实践。
